## 代码分析

### 1. `render()` 函数

`ui/display.py` 中的核心渲染函数，完全遵循接口文档的约定：

```python
def render(frame, result, fps=None) -> np.ndarray:
```

**参数**:
- `frame`: 原始 BGR 图像
- `result`: 识别结果字典，至少包含 `label` 和 `score`
- `fps`: 可选帧率显示

**返回**: 带标注的输出图像（不修改原图，内部先 `.copy()`）

### 2. 显示布局

渲染界面分为两个区域：

```
┌──────────────────────────────────────┐
│ [状态面板 - 浅灰背景]                 │
│  当前识别: 你好          ← 大号标题  │
│  置信度: 0.92  FPS: 28.5  类型: dynamic │
│                                      │
│──────────────────← 绿色/红色状态条── │
│                                      │
│ [摄像头画面 + 手部关键点/边框]       │
│                                      │
└──────────────────────────────────────┘
```

**关键特征**:

- **状态面板**：顶部 132 像素的浅灰色信息栏，无论识别结果如何都保持显示。
- **底部色条**：识别成功显示绿色、未识别显示红色，直观反馈系统状态。
- **中文渲染**：优先使用系统中文字体（Windows 的微软雅黑/黑体、Linux 的文泉驿/Noto），回退到 OpenCV 的 ASCII 渲染。

### 3. 中文字体渲染策略

```python
FONT_CANDIDATES = [
    "C:/Windows/Fonts/msyh.ttc",        # Windows 微软雅黑
    "C:/Windows/Fonts/simhei.ttf",      # Windows 黑体
    "/usr/share/fonts/.../wqy-microhei.ttc",  # Linux 文泉驿
    "/usr/share/fonts/.../NotoSansCJK-Regular.ttc",  # Linux Noto
]

def _draw_text(frame, text, position, font_size, color):
    if PIL is available:
        return _draw_text_pil(...)  # 中文渲染
    else:
        return _draw_text_opencv(...)  # ASCII 回退
```

- 使用 Pillow 在内存中渲染中文字体，再转回 OpenCV BGR 格式。
- 当 Pillow 不可用时（未安装或导入失败），自动降级为 OpenCV 原生文字渲染（仅 ASCII）。
- 文字位置和大小与底部渲染统一。

### 4. 手部检测可视化

```python
def _draw_detection(frame, bbox, landmarks):
    if bbox is valid:
        cv2.rectangle(frame, (x1,y1), (x2,y2), green, 2)
    for point in landmarks:
        cv2.circle(frame, point, 3, green, -1)
```

- 检测到手时绘制绿色边框和关键点。
- `_parse_point()` 支持归一化坐标（0–1 范围自动映射到图像尺寸）和绝对像素坐标两种格式。
- `_parse_bbox()` 带类型和数值校验，避免异常数据导致崩溃。

### 5. 最终主程序 `main.py`

4 号同学对 `main.py` 进行了大幅重写（从 69 行扩展到 131 行，实际重写超过该行数），将占位代码替换为真实集成：

**关键改动**：

1. **引入真实识别模块**：`import recognizer as recognizer_module`
2. **引入真实显示模块**：`from ui.display import render`
3. **完整的集成主循环**：

```python
while True:
    success, frame = camera.read()
    result = recognizer_module.recognize(frame)
    output = render(frame, result, fps=current_fps)
    cv2.imshow(WINDOW_NAME, output)
```

4. **FPS 平滑**：使用指数移动平均（EMA）`current_fps * 0.85 + instant_fps * 0.15`，显示更稳定。
5. **错误处理**：摄像头读取失败时显示灰色占位画面 + 错误提示，按 Q/ESC 退出。
6. **资源安全释放**：使用 `try/finally` 保证 `camera.release()` 和 `cv2.destroyAllWindows()` 一定执行。
7. **关闭钩子**：通过 `getattr(recognizer_module, "close", None)` 安全调用识别模块的清理函数。

**参数解析**：支持 `--camera`、`--width`、`--height`、`--fps`，其中 `--camera` 可设为 `auto` 自动探测。

## 设计评价

1. **完全符合接口规范**：`render(frame, result)` 签名与接口文档完全一致，输出可直接 `cv2.imshow()`。
2. **中文渲染完善**：考虑到了 OpenCV 不支持中文渲染的问题，使用 Pillow 方案解决，并有优雅降级。
3. **视觉反馈清晰**：状态面板 + 颜色条 + 关键点可视化，用户能清楚理解系统状态。
4. **健壮性强**：大量防御性编程，对空值、异常类型、缺失字段都有处理，不会因识别模块返回异常数据而崩溃。
5. **FPS 平滑算法**：EMA 比瞬时帧率更稳定，提供更好的使用体验。
