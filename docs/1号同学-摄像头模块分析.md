## 代码分析

### 1. `CameraService` 类

`camera/frame_provider.py` 中定义的摄像头封装，提供了完整的生命周期管理：

```python
class CameraService:
    def __init__(self, camera_index=0, width=640, height=480, fps=30):
    def open(self) -> bool:
    def read(self) -> Tuple[bool, Optional[np.ndarray]]:
    def release(self) -> None:
```

**设计要点**:

- **延迟打开**：`open()` 在首次 `read()` 时自动调用，而非构造函数中打开。这样可以延迟占用摄像头资源。
- **自动重连**：每次 `read()` 时如果摄像头未打开会自动调用 `open()`；摄像头中途断开后下次调用能自动尝试重连。
- **V4L2 后端**：强制使用 `cv2.CAP_V4L2`，在 Linux（包括树莓派）上更稳定。
- **分辨率与帧率控制**：统一设置为 640×480 @ 30 FPS，符合团队约定的接口规范。

### 2. `get_frame()` 快捷函数

```python
_default_camera = CameraService()

def get_frame():
    return _default_camera.read()
```

使用模块级单例，提供最简单的调用方式，无需关心 `CameraService` 的构造和生命周期。

### 3. `preview_camera()` 预览循环

```python
def preview_camera(camera=None, window_name="Camera Preview"):
```

核心预览循环，包含：
- 实时 FPS 计算与显示
- 绿色 "Camera OK" 状态提示
- 按 `q` 键退出

### 4. `probe_camera()` 自动探测

```python
def probe_camera(candidates=None, width=640, height=480, fps=30):
```

自动尝试多个候选设备（0–3 以及 /dev/video0–31），返回第一个能成功读取图像帧的设备索引。解决树莓派上摄像头设备号不确定的问题。

### 5. `preview_camera.py` 命令行入口

```python
python preview_camera.py              # 自动探测
python preview_camera.py --camera 0   # 指定设备
python preview_camera.py --no-window --output test.jpg  # 无界面抓拍
```

支持三种使用场景：自动探测、手动指定、无界面抓拍（适合树莓派无图形桌面时测试）。

### 6. 原始 `main.py`

初始版本包含：
- `mock_recognize()` — 占位识别函数，供联调使用
- `render()` — 占位显示逻辑
- `main()` — 使用 `probe_camera()` 检测摄像头 → 读取 → mock 识别 → 显示循环

这正是接口文档中约定的主流程：`camera.read() → recognize(frame) → render(frame, result) → imshow`。

1. **接口抽象合理**：`CameraService` 封装了摄像头生命周期，通过 `get_frame()` 快捷函数降低了队友的使用门槛。
2. **容错处理完善**：自动探测（`probe_camera`）、自动重连、空帧处理。
3. **跨平台兼容**：支持 Linux V4L2 和 Windows DirectShow。
