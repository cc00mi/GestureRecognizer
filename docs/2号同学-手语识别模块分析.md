## 代码分析

### 1. 统一识别接口 `recognizer.py`

```python
from dynamic_recognizer import DynamicHandGestureRecognizer

_default_recognizer = None

def recognize(frame):
    global _default_recognizer
    if _default_recognizer is None:
        _default_recognizer = DynamicHandGestureRecognizer(
            score_threshold=0.65, stable_count=4)
    return _default_recognizer.recognize(frame)
```

**设计要点**:

- **单例模式**：使用模块级全局变量延迟初始化，避免重复创建 MediaPipe 实例（加载耗时）。
- **接口对齐**：`recognize(frame)` 完全遵循接口文档约定的签名，方便 4 号同学直接调用。
- **参数调优**：`score_threshold=0.65`（置信度阈值）、`stable_count=4`（稳定次数），在灵敏度和误报之间取得平衡。

### 2. 动态手语识别核心 `dynamic_recognizer.py`

`DynamicHandGestureRecognizer` 类实现了完整的实时识别流程：

```python
class DynamicHandGestureRecognizer:
    def __init__(self, sequence_length=30, score_threshold=0.7, stable_count=5, ...):
    def recognize(self, frame):
    def close(self):
```

**核心流程**：

```
frame → BGR→RGB → MediaPipe Hands → 关键点归一化 →
  存入序列队列 → 满30帧 → TFLite 推理 → 置信度过滤 →
  稳定计数 → 输出结果
```

**三个关键机制**：

1. **时序缓冲**：`deque(maxlen=30)` 维护最近 30 帧的双手关键点序列，实现动态动作识别而非单帧静态手势。
2. **置信度阈值**：低于 0.65 的结果被丢弃，避免低质量识别。
3. **稳定计数**：同一结果连续出现 4 次才输出，消除单帧误识别和动作过渡期的抖动。

**双手特征提取** `make_frame_features()`：

```python
def make_frame_features(results):
    # 左手 21点 × XYZ = 63维
    # 右手 21点 × XYZ = 63维
    # 合计 126维 / 帧
    return frame_features['Left'] + frame_features['Right']
```

- 某个手未检测到时对应 63 维补零，容错性强。
- 使用手腕关键点(landmark[0])作为参考点做归一化，消除手在画面中位置的影响。

### 4. 数据集提取 `extract_dynamic_dataset.py`

```python
python extract_dynamic_dataset.py --dataset vs-dataset --sequence_length 30 --max_num_hands 2
```

**核心功能**：

- 遍历按类别分文件夹存放的视频数据集
- 逐帧提取 MediaPipe 双手关键点
- 使用线性插值重采样为固定长度（30帧）
- 输出 CSV 文件（label_id + 序列展开）

**重采样方法**：使用 `np.linspace` 均匀采样指定数量的帧，适配变长视频。

### 5. 模型训练 `train_dynamic_gesture_classifier.py`

```python
python train_dynamic_gesture_classifier.py --epochs 100 --feature_dim 126
```

**模型架构**（Keras Sequential）：

```
Input(30, 126)
  → Conv1D(64, 3) + BN + ReLU
  → Conv1D(64, 3) + BN + ReLU
  → MaxPool1D(2) + Dropout(0.3)
  → Conv1D(128, 3) + BN + ReLU
  → GlobalAveragePooling1D + Dropout(0.3)
  → Dense(64, ReLU)
  → Dense(num_classes, Softmax)
```

**设计特点**：

- 两层 Conv1D 提取时序特征，GlobalAveragePooling1D 减少参数量。
- 使用 EarlyStopping（patience=15）防止过拟合。
- 分层验证集拆分：按类别比例抽取 20% 作为验证集，保证每个类别都有验证样本。
- 输出 H5 + TFLite 双格式。

## 技术方案选型

1. **动态识别而非静态**：采集 30 帧序列识别完整手语动作，而不是单帧静态手势，能区分"再见"（挥手动作）、"没关系"（摆手动作）等动态类别。
2. **MediaPipe Hands**：相比传统视觉方法，21 个关键点检测稳定，且跨平台性能好。树上运行效率可接受。
3. **TFLite 端侧推理**：模型经过 TensorFlow Lite 转换后约 289KB，可在树莓派上实时运行，无需 GPU。
4. **双手支持**：同时检测左右手关键点，支持需要双手配合的手语类别。

## 设计评价

1. **接口清晰**：对外只暴露 `recognize(frame)`，队友无需理解内部实现。
2. **时序处理完善**：使用队列缓存 + 稳定计数的策略有效减少了实时识别中的误报和抖动。
3. **可扩展**：支持增加新类别（只需补充视频数据、重新提取和训练）。
4. **容错性好**：单手未检测到自动补零，空帧直接返回"未识别"。
