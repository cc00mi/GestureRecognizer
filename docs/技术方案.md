# 技术方案

做一个基于树莓派的固定手语识别系统：用户对着摄像头做手势，系统实时识别手势含义。

- 硬件：`树莓派 + 摄像头 + 电脑显示器/电脑远程查看`
- 算法：`MediaPipe 检测手 -> 提取关键点 -> 简单分类`
- 显示：识别结果直接显示成中文
- 不要求连续自然语言翻译
- 不要求大数据集
- 优先使用现成开源示例改

最适合参考的开源项目：

- [kinivi/hand-gesture-recognition-mediapipe](https://github.com/kinivi/hand-gesture-recognition-mediapipe)

这个项目很适合你们，因为它已经把：

- 摄像头读取
- MediaPipe 手部关键点
- 简单分类
- 实时识别

都做了，你们主要是“改类别、改显示、改成中文”。

## 1号同学：树莓派和摄像头

任务目标：让设备能跑摄像头画面。

具体做什么：

1. 安装树莓派系统
2. 连接摄像头
3. 测试能否看到实时画面
4. 连接显示器，或者让电脑远程访问树莓派桌面
5. 安装 Python、OpenCV、MediaPipe

最简单交付：

- 树莓派成功开机
- 摄像头能打开
- 能在屏幕上显示视频画面

建议直接做的程序：

- 运行一个最简单的摄像头预览脚本

参考资料：

- Raspberry Pi Camera 文档  
  [https://www.raspberrypi.com/documentation/computers/camera_software.html](https://www.raspberrypi.com/documentation/computers/camera_software.html)
- Raspberry Pi 远程访问  
  [https://www.raspberrypi.com/documentation/computers/remote-access.html](https://www.raspberrypi.com/documentation/computers/remote-access.html)

和别人怎么对接：

- 给 2 号同学提供“摄像头画面能正常读取”的环境
- 只需要保证别人运行程序时，摄像头能用

## 2号同学：手势识别

任务目标：把手势识别出来。

具体做什么：

1. 下载并运行这个开源项目  
   [https://github.com/kinivi/hand-gesture-recognition-mediapipe](https://github.com/kinivi/hand-gesture-recognition-mediapipe)
2. 先不要自己重新造模型，先跑通原项目
3. 把里面的类别改成你们需要的几个手势
4. 如果来不及训练，就先做少量样本训练
5. 输出识别结果，比如“你好”“谢谢”

最简单交付：

- 摄像头中伸出手后，程序能输出一个类别名
- 至少识别 `5~10` 个手势

最简单技术设计：

- 用 `MediaPipe` 找手部 21 个关键点
- 用开源项目里的简单分类器判断属于哪个手势
- 输出一个文字标签

参考资料：

- MediaPipe  
  [https://mediapipe.org/](https://mediapipe.org/)
- 开源样例  
  [https://github.com/kinivi/hand-gesture-recognition-mediapipe](https://github.com/kinivi/hand-gesture-recognition-mediapipe)

和别人怎么对接：

- 给 4 号同学一个输出结果
- 格式非常简单就行：

```python
label = "谢谢"
```

或者：

```python
result = {"label": "谢谢", "score": 0.92}
```

## 3号同学：数据采集和类别整理

任务目标：准备最基础的数据，不追求很大。

具体做什么：

1. 先定 `5~10` 个手势类别
2. 每个类别找同学录一点样本
3. 样本不用很多，每类先录 `20~30` 次
4. 整理类别名称表
5. 记录哪些手势容易混淆

最简单交付：

- 一个类别清单
- 一小批训练样本
- 一个“标签对应表”

建议类别：

- `你好`
- `谢谢`
- `喝水`
- `吃饭`
- `回家`
- `1`
- `2`
- `3`
- `4`
- `5`

最简单标签表：

```json
{
  "0": "你好",
  "1": "谢谢",
  "2": "喝水",
  "3": "吃饭",
  "4": "回家"
}
```

## 4号同学：显示界面

任务目标：把识别结果显示出来。

具体做什么：

1. 写一个最简单的界面
2. 界面上显示：
   - 摄像头画面
   - 当前识别结果
3. 如果不会网页，就直接用 OpenCV 窗口显示文字
4. 如果会一点 Python GUI，可以用 `Tkinter`
5. 先不做复杂历史记录、语音播报

最简单交付：

- 窗口里能看到摄像头画面
- 窗口上能显示“当前识别：谢谢”
