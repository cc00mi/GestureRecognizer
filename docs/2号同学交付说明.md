## 4. 识别接口

4 号同学或主程序调用方式：

```python
from dynamic_recognizer import recognize

result = recognize(frame)
```

参数说明：

```python
frame: np.ndarray
```

要求传入 OpenCV 摄像头读取到的 BGR 图像帧。

返回格式：

```python
{
    "label_id": 0,
    "label": "你好",
    "score": 0.93,
    "type": "dynamic"
}
```

没有稳定识别到完整手语时返回：

```python
{
    "label_id": -1,
    "label": "未识别",
    "score": 0.0,
    "type": "dynamic"
}
```

界面模块只需要读取：

```python
result["label"]
result["score"]
```

即可显示当前识别结果。

## 5. 双手动态识别方案

当前模型使用双手关键点序列作为输入。

每一帧特征格式：

```text
Left hand  21点 * xyz = 63维
Right hand 21点 * xyz = 63维
合计 126维
```

如果某一只手没有检测到，则对应 63 维补 0。

每个视频会被重采样为 30 帧：

```text
30帧 * 126维
```

这样可以识别动态动作，而不是只识别单帧静态手势。

相关代码：

```text
extract_dynamic_dataset.py
train_dynamic_gesture_classifier.py
dynamic_recognizer.py
model/dynamic_gesture_classifier/
```

## 6. 数据集结构

3 号同学提供或整理的视频数据集按类别文件夹存放：

```text

    你好1.mp4
  再见/
    再见.mp4
  对不起/
  我爱你/
  没关系/

```

要求：

- 一个视频只包含一个完整手语动作。
- 手尽量完整出现在画面中。
- 背景、距离、光照尽量保持一致。
- 双手动作类别需要两只手都尽量入镜。

## 10. 实时识别稳定参数

当前主程序中动态识别参数：

```python
dynamic_recognizer = DynamicHandGestureRecognizer(
    score_threshold=0.65,
    stable_count=4,
)
```

参数含义：

- `score_threshold`：置信度阈值，低于该值不输出。
- `stable_count`：连续稳定次数，同一个结果连续出现足够多次后才输出。

调参建议：

- 如果经常没有结果：降低 `score_threshold`，例如 `0.55`。
- 如果误识别较多：提高 `stable_count`，例如 `5` 或 `6`。
- 不建议只靠提高 `score_threshold` 抑制误识别，动态动作切换时更适合用 `stable_count` 控制。

## 11. 给 4 号同学的最小对接示例

```python
import cv2 as cv
from dynamic_recognizer import recognize

cap = cv.VideoCapture(0)

while True:
    success, frame = cap.read()
    if not success:
        break

    result = recognize(frame)

    if result["label_id"] >= 0:
        text = f"{result['label']} {result['score']:.2f}"
    else:
        text = "未识别"

    cv.putText(frame, text, (10, 40), cv.FONT_HERSHEY_SIMPLEX,
               1.0, (0, 255, 0), 2, cv.LINE_AA)
    cv.imshow("Sign Recognition", frame)

    if cv.waitKey(10) == 27:
        break

cap.release()
cv.destroyAllWindows()
```

## 12. 目前限制和后续优化

当前主要限制：

- 训练样本数量仍偏少。
- 部分类别容易受动作过渡影响产生误识别。
- 如果实时动作和训练视频的角度、距离、速度差异较大，置信度会下降。
- OpenCV 默认 `putText` 对中文显示支持不好，中文标签更适合由 4 号同学使用支持中文字体的方式绘制。

后续优化建议：

1. 每个类别补充更多视频，尤其是容易混淆的类别。
2. 录制时保持双手完整入镜。
3. 增加不同距离、不同速度、不同人员的样本。
4. 对动作开始和结束姿势做统一要求。
5. 根据演示效果微调 `score_threshold` 和 `stable_count`。
